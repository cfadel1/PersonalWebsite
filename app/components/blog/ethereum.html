Ethereum Smart Contracts

May 4, 2018
The purpose of this blog is to briefly review some concepts regarding Ethereum smart contracts. This knowledge should help enhance the reader's understanding of the blog “Ethereum Smart Contract Security Vulnerabilities”. The notions that will be described in this blog are the blockchain technology, Ethereum, Ethereum Virtual Machine, ether, smart contracts and gas.

What is blockchain technology?

Blockchain is a data structure similar to an immutable linked-list. A simple way of defining blockchain is to describe it as a digital ledger of records that are arranged in chunks of data called blocks. The blockchain maintains a continuously-growing list of blocks.
The blocks are linked to one another through a hash function, otherwise known as cryptographic validation. For instance, Bitcoin uses the SHA-256 hash function.
Some of the main characteristics for the blockchain are:
  • Append only - Elements only get added to the end of the list
  • Public - All users hold the same copy of the blockchain and it is public to everyone to look at
  • Immutable - All changes that happen on the blockchain remain on the blockchain forever and cannot be edited, updated or deleted
  • Decentralized - Peer to peer platform
This technology can be used in many different application domains such as finance, IoT, security, public service, cryptocurrency and many others.
A more detailed explanation of the blockchain will be added in our discussion of the security vulnerabilities of smart contracts related to the blockchain.

Mining

Mining is the process of adding a block and verifying the validity of transactions through a Proof of Work (PoW) or other consensus protocols.

What is Ethereum?

Ethereum is a blockchain distributed computing platform. The main difference between Ethereum from other cryptocurrency-based platform is that it does not only serve as a payment system but also as a computing platform. Some refer to it as The World's Computer due to its availability to everyone. Others refer to it as the future of internet due to its transparency.

Ethereum Virtual Machine

Ethereum provides a decentralized turing-complete virtual machine. Turing-complete essentially means you can write programs known as contracts that can solve any reasonable computational problem. The virtual machine is known as the Ethereum Virtual Machine or for short EVM. The EVM is a run time environment for smart contracts. Smart contracts are described in the next section. The EVM has a 256-bit register stack which is designed to run the code. It is completely isolated from the network, filesystem and the host computer system. Every single Ethereum node in the network runs an EVM implementation and executes the same instructions.

What is Ether?

Sometimes, ether and Ethereum are interchangeable but in this blog, ether will represent a cryptocurrency whose blockchain is generated by the Ethereum platform. At the time of writing, an ether has a value of $789.75.

What are smart contracts?

Smart contracts are a collection of code and data that reside on a specific address on the Ethereum blockchain. Smart contracts contain the code function that enable the interaction with other contracts, make decisions, store data and send ether between users. They are defined by their creators. Their execution and the services they offer is provided by the EVM itself. They also hold the characteristic of immutability since they are located on the blockchain. This means once a contract has been deployed on the blockchain, there is no way of editing it or deleting it. This increases the importance of the attention to detail given when creating a contract. If a contract has a bug in it or a security vulnerability then the contract is considered to be useless. In other words, smart contracts are like normal contracts but are digital. The advantage of having smart contracts on the blockchain is that it is immutable and can resemble a stable law structure.

How can they be used?

Smart contracts can be used to replace real estate contracts, trust funds, wills, Airbnb, Uber and many other services. Any form of contract can be replaced with a digital one. A digital contract will be copied on all the nodes and will be immutable.

What is solidity

Solidity is a contract-oriented programming language for writing smart contracts. It runs on the EVM and it is read sequentially. Solidity is statically typed and the type of each variable needs to be specified at compile-time. The language is easy to learn and similar to JavaScript. A really good online tutorial is CryptoZombie. In this tutorial, the reader will learn how to build a game a game on Ethereum. It is designed for beginners to Solidity but it also assumes that the user has some experience programming in another language. The tutorial also explains concepts of real Decentralized Applications (DApps) such as contract ownership, gas costs, code optimization and security. In addition, it explains the concept of the ERC721 standard and cryptocollectible assets. Lastly, it also includes a module on Front-End development where it introduces frameworks like Infura and Metamas and Web3.js in order to have the web browser communicate with the blockchain.

What is gas?

Gas is known as the fuel an Ethereum decentralized application runs on. In solidity, users have to pay every time they execute a function using the currency of gas. Users pay for gas with Ether.

How much gas is required to execute a function?

It all depends on its complexity in terms of function’s logic. Each individual operation has a gas cost based on how much computing resources will be required to perform that operation. For example, writing to storage is much more expensive than adding two integers. This is why code optimization is much more important in solidity than in other programming languages.
